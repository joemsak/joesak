Testing Workflow

Red. Green. Refactor.

  1. Start with a failing test. A compilation error (missing file, undefined class) counts as a failure.
  2. Write just enough production code to pass the test. Even if it's a hard-coded return value.
  3. Refactor the production code relentlessly. Repeat for the test code. Do not overlook your tests when you're refactoring.

Following this workflow diligently and with discipline guarantees that you will get that programmer's rush every few minutes.
